# Compression Levels

## Safe
- Maximum semantic retention.
- Explicit constraints and context.
- Best for risk-sensitive execution.

## Balanced
- Strong structural compaction.
- Removes redundancy while preserving intent.
- Default for GPT/Codex/Hermes.

## Aggressive
- Strong abbreviation and denser seed usage.
- Must preserve safety + goal + output.
- Can trigger warnings when nuance risk increases.
